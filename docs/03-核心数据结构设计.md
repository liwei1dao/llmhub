# LLMHub 核心数据结构设计

- 版本：v0.1
- 日期：2026-04-22
- 数据库：PostgreSQL 16 + TimescaleDB 扩展

---

## 1. 数据库分域

按业务域划分 schema，方便后期拆库：

| Schema | 用途 |
|--------|------|
| `iam` | 用户、认证、API Key |
| `wallet` | 余额、充值、消费 |
| `pool` | 上游账号池、凭证引用 |
| `catalog` | Provider、Model、定价 |
| `metering` | 调用流水、用量聚合（TimescaleDB） |
| `audit` | 审计日志（追加型） |

## 3. 核心表 DDL

### 3.1 IAM 域

```sql
-- 用户
CREATE TABLE iam.users (
    id              BIGSERIAL PRIMARY KEY,
    email           VARCHAR(255) UNIQUE,
    phone           VARCHAR(32) UNIQUE,
    password_hash   VARCHAR(128) NOT NULL,     -- argon2id
    display_name    VARCHAR(64),
    status          VARCHAR(16) NOT NULL DEFAULT 'active',
                    -- active / suspended / banned / deleted
    realname_level  SMALLINT DEFAULT 0,         -- 0 未认证 / 1 基础 / 2 企业
    risk_score      SMALLINT DEFAULT 0,         -- 0-100，风险评分
    qps_limit       INT DEFAULT 10,
    daily_spend_limit_cents BIGINT DEFAULT 100000, -- 日消费上限
    metadata        JSONB NOT NULL DEFAULT '{}'::jsonb,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    last_login_at   TIMESTAMPTZ
);

CREATE INDEX idx_users_status ON iam.users(status) WHERE status != 'active';
CREATE INDEX idx_users_risk ON iam.users(risk_score) WHERE risk_score > 60;

-- 用户 API Key
CREATE TABLE iam.api_keys (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT NOT NULL REFERENCES iam.users(id),
    prefix          VARCHAR(32) NOT NULL,       -- sk-llmh-1a2b
    key_hash        VARCHAR(128) NOT NULL,      -- SHA-256
    name            VARCHAR(64),                 -- 用户自己起的名字
    scopes          TEXT[] NOT NULL DEFAULT '{}', -- 允许的模型或能力
    ip_allowlist    INET[] NOT NULL DEFAULT '{}',
    status          VARCHAR(16) NOT NULL DEFAULT 'active',
                    -- active / revoked / expired
    usage_cap_cents BIGINT,                      -- 该 Key 消费上限，null 表示无限
    used_cents      BIGINT NOT NULL DEFAULT 0,
    last_used_at    TIMESTAMPTZ,
    expires_at      TIMESTAMPTZ,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE UNIQUE INDEX uq_api_keys_hash ON iam.api_keys(key_hash);
CREATE INDEX idx_api_keys_user ON iam.api_keys(user_id, status);
CREATE INDEX idx_api_keys_prefix ON iam.api_keys(prefix) WHERE status = 'active';

-- 登录会话
CREATE TABLE iam.sessions (
    id              UUID PRIMARY KEY,
    user_id         BIGINT NOT NULL REFERENCES iam.users(id),
    token_hash      VARCHAR(128) NOT NULL,
    user_agent      TEXT,
    ip              INET,
    expires_at      TIMESTAMPTZ NOT NULL,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    revoked_at      TIMESTAMPTZ
);
```

### 3.2 Wallet 域

```sql
-- 钱包账户（一个用户一个主账户，未来可支持多币种/子账户）
CREATE TABLE wallet.accounts (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT NOT NULL REFERENCES iam.users(id),
    currency        VARCHAR(8) NOT NULL DEFAULT 'CNY',
    balance_cents   BIGINT NOT NULL DEFAULT 0,
    frozen_cents    BIGINT NOT NULL DEFAULT 0,  -- 预扣费冻结金额
    total_recharged_cents  BIGINT NOT NULL DEFAULT 0,
    total_spent_cents      BIGINT NOT NULL DEFAULT 0,
    version         INT NOT NULL DEFAULT 0,      -- 乐观锁
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),

    CONSTRAINT ck_balance_nonneg CHECK (balance_cents >= 0),
    CONSTRAINT ck_frozen_nonneg  CHECK (frozen_cents >= 0 AND frozen_cents <= balance_cents)
);

CREATE UNIQUE INDEX uq_wallet_user ON wallet.accounts(user_id, currency);

-- 交易流水（所有余额变动必须通过这张表）
CREATE TABLE wallet.transactions (
    id              BIGSERIAL PRIMARY KEY,
    account_id      BIGINT NOT NULL REFERENCES wallet.accounts(id),
    type            VARCHAR(32) NOT NULL,
                    -- recharge / spend / freeze / unfreeze / refund / adjust
    amount_cents    BIGINT NOT NULL,             -- 正：入账；负：出账
    balance_after   BIGINT NOT NULL,             -- 交易后余额
    related_id      VARCHAR(64),                  -- 关联的调用/充值/退款 ID
    related_type    VARCHAR(32),                  -- call / recharge / invoice ...
    memo            TEXT,
    metadata        JSONB NOT NULL DEFAULT '{}'::jsonb,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_txn_account_created ON wallet.transactions(account_id, created_at DESC);
CREATE INDEX idx_txn_related ON wallet.transactions(related_type, related_id);

-- 充值订单
CREATE TABLE wallet.recharges (
    id              BIGSERIAL PRIMARY KEY,
    order_no        VARCHAR(64) UNIQUE NOT NULL,
    user_id         BIGINT NOT NULL REFERENCES iam.users(id),
    amount_cents    BIGINT NOT NULL,
    channel         VARCHAR(32) NOT NULL,        -- alipay / wechat / stripe / manual
    channel_order_id VARCHAR(128),
    status          VARCHAR(16) NOT NULL DEFAULT 'pending',
                    -- pending / paid / failed / refunded / cancelled
    paid_at         TIMESTAMPTZ,
    metadata        JSONB NOT NULL DEFAULT '{}'::jsonb,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 发票（V1）
CREATE TABLE wallet.invoices (
    id              BIGSERIAL PRIMARY KEY,
    user_id         BIGINT NOT NULL REFERENCES iam.users(id),
    invoice_no      VARCHAR(64) UNIQUE,
    amount_cents    BIGINT NOT NULL,
    title           VARCHAR(256),
    tax_no          VARCHAR(64),
    email           VARCHAR(255),
    status          VARCHAR(16) NOT NULL DEFAULT 'pending',
    issued_at       TIMESTAMPTZ,
    file_url        TEXT,
    metadata        JSONB NOT NULL DEFAULT '{}'::jsonb,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### 3.3 Catalog 域（供应商 & 模型）

```sql
-- Provider 供应商
CREATE TABLE catalog.providers (
    id              VARCHAR(32) PRIMARY KEY,     -- volc / dashscope / deepseek / anthropic
    display_name    VARCHAR(64) NOT NULL,
    base_url        TEXT NOT NULL,
    auth_mode       VARCHAR(32) NOT NULL,         -- bearer / ak_sk / signed_token
    protocol_family VARCHAR(32) NOT NULL,         -- openai_compat / anthropic / custom
    status          VARCHAR(16) NOT NULL DEFAULT 'active',
                    -- active / paused / deprecated
    config          JSONB NOT NULL DEFAULT '{}'::jsonb,  -- adapter 特定配置
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 平台对外暴露的"逻辑模型"
CREATE TABLE catalog.models (
    id              VARCHAR(64) PRIMARY KEY,     -- llmhub 对外模型 ID，如 "gpt-4o-class"
    display_name    VARCHAR(128) NOT NULL,
    category        VARCHAR(32) NOT NULL,         -- chat / embedding / image / audio
    capabilities    TEXT[] NOT NULL DEFAULT '{}', -- ['tool_use', 'vision', 'json_mode']
    context_window  INT,
    max_output      INT,
    is_public       BOOLEAN NOT NULL DEFAULT TRUE,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- 逻辑模型 → Provider 具体模型的映射
CREATE TABLE catalog.model_mappings (
    id              BIGSERIAL PRIMARY KEY,
    model_id        VARCHAR(64) NOT NULL REFERENCES catalog.models(id),
    provider_id     VARCHAR(32) NOT NULL REFERENCES catalog.providers(id),
    upstream_model  VARCHAR(128) NOT NULL,        -- 如 ep-xxxxx / doubao-pro-32k
    priority        SMALLINT NOT NULL DEFAULT 10, -- 路由优先级，越小越优先
    status          VARCHAR(16) NOT NULL DEFAULT 'active',
    notes           TEXT,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE UNIQUE INDEX uq_model_mapping ON catalog.model_mappings(model_id, provider_id, upstream_model);

-- 上游成本（用于毛利核算）+ 对外零售价
CREATE TABLE catalog.pricing (
    id              BIGSERIAL PRIMARY KEY,
    model_id        VARCHAR(64) NOT NULL REFERENCES catalog.models(id),
    provider_id     VARCHAR(32) REFERENCES catalog.providers(id), -- NULL = 平台统一零售价
    kind            VARCHAR(16) NOT NULL,         -- retail / wholesale
    input_per_1k_cents   DECIMAL(12, 4) NOT NULL,
    output_per_1k_cents  DECIMAL(12, 4) NOT NULL,
    effective_from  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    effective_until TIMESTAMPTZ,
    notes           TEXT
);

CREATE INDEX idx_pricing_lookup ON catalog.pricing(model_id, provider_id, kind, effective_from DESC);
```

### 3.5 Metering 域（调用流水，时序）

```sql
-- 调用流水（TimescaleDB Hypertable）
CREATE TABLE metering.call_logs (
    id              UUID NOT NULL,
    ts              TIMESTAMPTZ NOT NULL,          -- 调用时间
    user_id         BIGINT NOT NULL,
    api_key_id      BIGINT NOT NULL,
    request_id      VARCHAR(64) NOT NULL,          -- 对外暴露的请求 ID
    model_id        VARCHAR(64) NOT NULL,
    provider_id     VARCHAR(32) NOT NULL,
    pool_account_id BIGINT NOT NULL,
    upstream_model  VARCHAR(128) NOT NULL,
    status          VARCHAR(16) NOT NULL,          -- success / failed / partial
    error_code      VARCHAR(64),
    duration_ms     INT NOT NULL,
    ttfb_ms         INT,                           -- 首 Token 延迟
    tokens_in       INT NOT NULL DEFAULT 0,
    tokens_out      INT NOT NULL DEFAULT 0,
    cost_wholesale_cents DECIMAL(16, 6) NOT NULL DEFAULT 0,  -- 上游成本
    cost_retail_cents    DECIMAL(16, 6) NOT NULL DEFAULT 0,  -- 用户侧计费
    retry_count     SMALLINT NOT NULL DEFAULT 0,
    fallback_chain  JSONB,                         -- 实际走过的 provider 链
    metadata        JSONB NOT NULL DEFAULT '{}'::jsonb,
    PRIMARY KEY (id, ts)
);

SELECT create_hypertable('metering.call_logs', 'ts', chunk_time_interval => INTERVAL '1 day');

CREATE INDEX idx_calls_user_ts ON metering.call_logs(user_id, ts DESC);
CREATE INDEX idx_calls_provider_ts ON metering.call_logs(provider_id, ts DESC);
CREATE INDEX idx_calls_pool_account_ts ON metering.call_logs(pool_account_id, ts DESC);
CREATE INDEX idx_calls_status ON metering.call_logs(status, ts DESC) WHERE status != 'success';
CREATE INDEX idx_calls_request_id ON metering.call_logs(request_id);

-- 用户级日聚合（continuous aggregate）
CREATE MATERIALIZED VIEW metering.user_daily_usage
WITH (timescaledb.continuous) AS
SELECT
    time_bucket('1 day', ts) AS day,
    user_id,
    model_id,
    provider_id,
    COUNT(*)                              AS calls_total,
    COUNT(*) FILTER (WHERE status='success')  AS calls_success,
    COUNT(*) FILTER (WHERE status='failed')   AS calls_failed,
    SUM(tokens_in)                        AS tokens_in,
    SUM(tokens_out)                       AS tokens_out,
    SUM(cost_wholesale_cents)             AS cost_wholesale,
    SUM(cost_retail_cents)                AS cost_retail,
    AVG(ttfb_ms)                          AS avg_ttfb,
    MAX(duration_ms)                      AS max_duration
FROM metering.call_logs
GROUP BY 1, 2, 3, 4;

-- 池账号日聚合（诊断 + 对账用）
CREATE MATERIALIZED VIEW metering.pool_daily_usage
WITH (timescaledb.continuous) AS
SELECT
    time_bucket('1 day', ts) AS day,
    pool_account_id,
    provider_id,
    COUNT(*)                              AS calls_total,
    COUNT(*) FILTER (WHERE status='success')  AS calls_success,
    SUM(tokens_in)                        AS tokens_in,
    SUM(tokens_out)                       AS tokens_out,
    SUM(cost_wholesale_cents)             AS cost_wholesale
FROM metering.call_logs
GROUP BY 1, 2, 3;

-- 余额冻结表（与 call 一一对应）
CREATE TABLE metering.balance_holds (
    request_id      VARCHAR(64) PRIMARY KEY,
    user_id         BIGINT NOT NULL,
    account_id      BIGINT NOT NULL,              -- wallet.accounts.id
    amount_cents    BIGINT NOT NULL,
    status          VARCHAR(16) NOT NULL,          -- held / released / settled
    expires_at      TIMESTAMPTZ NOT NULL,          -- 超时自动释放
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    settled_at      TIMESTAMPTZ
);

CREATE INDEX idx_holds_expiry ON metering.balance_holds(status, expires_at) WHERE status = 'held';

-- 对账差异记录
CREATE TABLE metering.reconciliation (
    id              BIGSERIAL PRIMARY KEY,
    day             DATE NOT NULL,
    provider_id     VARCHAR(32) NOT NULL,
    pool_account_id BIGINT NOT NULL,
    platform_cost_cents  DECIMAL(16, 6) NOT NULL,
    upstream_bill_cents  DECIMAL(16, 6),
    diff_cents      DECIMAL(16, 6),
    diff_ratio      DECIMAL(8, 4),
    status          VARCHAR(16) NOT NULL,         -- ok / warn / alert
    notes           TEXT,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE UNIQUE INDEX uq_recon ON metering.reconciliation(day, pool_account_id);
```

## 4. 关键状态机

### 4.1 PoolAccount 状态机

```
                    ┌──录入──┐
                    │        │
                    ▼        │
                 [warmup]    │  (仅 T3 新号)
                    │        │
                    │ 养号完成 │
                    ▼        │
   ┌────────────▶[active]◀──┘
   │                │
   │  健康度↑       │  5xx 连续/429
   │                ▼
   │           [cooling]  ──指数退避60s/300s/1800s──┐
   │                │                                │
   │ 超时恢复        │   持续失败                     │
   └────────────────┘         ▼                      │
                         [rate_limited]              │
                              │                      │
                              │  持续失败            │
                              ▼                      │
                         [quarantine]  ◀────────────┘
                              │
                  人工或系统判定：
                  可恢复 → active
                  不可恢复 ↓
                              │
                              ▼
                         [archived]
                              ▲
                              │
                         [depleted]   (余额耗尽)
```

**迁移触发条件**：

| From | To | 条件 |
|------|----|------|
| warmup | active | 养号期满（默认 7 天）且调用成功 ≥ 20 次 |
| active | cooling | 连续 3 次 5xx 或 1 次 429 |
| cooling | active | TTL 到期且健康度 ≥ 40 |
| cooling | rate_limited | TTL 到期后仍 429 |
| rate_limited | quarantine | 连续 3 个 cooling 周期仍失败 |
| quarantine | active | 运营人工确认 + 健康度重置 |
| quarantine | archived | 运营人工确认不可恢复 |
| * | depleted | `quota_used >= quota_total` |
| depleted | archived | 过了账期或无法续费 |

### 4.2 WalletTransaction 生命周期（预扣费）

```
调用开始：
  INSERT metering.balance_holds (status='held')
  UPDATE wallet.accounts SET frozen_cents += amount WHERE version = v
  INSERT wallet.transactions (type='freeze', amount=-freeze)

调用成功：
  UPDATE metering.balance_holds SET status='settled'
  UPDATE wallet.accounts
    SET balance_cents -= actual_cost,
        frozen_cents  -= freeze,
        total_spent_cents += actual_cost
  INSERT wallet.transactions (type='unfreeze', amount=+freeze)
  INSERT wallet.transactions (type='spend', amount=-actual_cost)

调用失败：
  UPDATE metering.balance_holds SET status='released'
  UPDATE wallet.accounts SET frozen_cents -= freeze
  INSERT wallet.transactions (type='unfreeze', amount=+freeze)

超时保护（定时任务）：
  查找 status='held' AND expires_at < NOW() 的 holds
  按"调用失败"路径释放
```

## 5. 关键索引和性能考量

| 场景 | 表 | 索引 |
|------|----|----|
| API Key 鉴权（每次请求） | `iam.api_keys` | `uq_api_keys_hash` + Redis 前置缓存 |
| 调度器挑号（每次请求） | `pool.accounts` | `idx_pool_accounts_selector` 覆盖 provider/status/tier/health，实际生产从 Redis 拉 |
| 用户查账单 | `metering.user_daily_usage`（聚合视图） | 避免扫全量 call_logs |
| 账号诊断 | `metering.call_logs` | `idx_calls_pool_account_ts`，按账号 + 时间 |
| 对账 | `pool_daily_usage` + `reconciliation` | 日粒度聚合后比对 |

**容量估算（MVP）**：

- call_logs：假设 100 QPS × 86400s × 365 天 ≈ 30 亿条/年。TimescaleDB 按日分片 + 压缩后压力可控。30 天后启用压缩，保留 90 天明细，更早的归档到对象存储。
- 用户表、账号池表：万级别，单库无压力。
- wallet.transactions：每次调用至少 2-3 条，和 call_logs 同量级，考虑提前做归档策略。

### 6.1 敏感字段清单

| 字段 | 处理方式 |
|------|----------|
| `iam.users.password_hash` | Argon2id，永不明文 |
| `iam.api_keys.key_hash` | SHA-256，明文只在创建时一次性返回 |
| `pool.accounts.metadata.credentials` | **禁止存放**，凭证全走 Vault |
| `pool.api_keys.vault_ref` | 只存 Vault 引用，不存 Key 本体 |
| `call_logs` 的 prompt/completion | **默认不存**，仅存 request_id 和 token 计数 |
| `audit.logs.payload` | 入库前字段级脱敏（关键字白名单） |

### 6.2 读写权限分离

```
应用账号：
  - app_gateway      (读所有业务表 + 写 call_logs/balance_holds)
  - app_account      (读写 iam.* / wallet.*)
  - app_pool_admin   (读写 pool.*)
  - app_readonly     (BI/分析用，只读聚合视图)
```

### 6.3 数据导出与删除（合规）

- 用户注销：逻辑删除，30 天后物理清理（除法规要求保留的流水）
- 调用内容日志若启用，TTL 7 天强制清理
- 账单凭证按合规要求保留 ≥ 5 年
