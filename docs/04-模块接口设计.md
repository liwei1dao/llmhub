# LLMHub 模块接口设计

- 版本：v0.1
- 日期：2026-04-22

---

## 1. 接口概览

分三类：

| 类别 | 协议 | 使用者 |
|------|------|--------|
| **对外业务 API** | HTTP/JSON + SSE | 终端用户、SDK、第三方应用 |
| **管理 API** | HTTP/JSON | 运营后台、用户控制台 |
| **内部服务 API** | gRPC / NATS 事件 | 各微服务之间 |

---

## 2. 对外业务 API（与大模型调用相关）

**基础 URL**：`https://api.llmhub.io`
**鉴权**：`Authorization: Bearer sk-llmh-xxxx`
**版本**：URL 路径 `/v1`、`/anthropic/v1` 双协议栈

### 2.1 OpenAI 兼容路径

#### 2.1.1 `POST /v1/chat/completions`

完全兼容 OpenAI Chat Completions 接口。

**请求**（示例）：

```json
{
  "model": "gpt-4o-class",
  "messages": [
    {"role": "system", "content": "..."},
    {"role": "user", "content": "..."}
  ],
  "stream": true,
  "temperature": 0.7,
  "max_tokens": 2000,
  "tools": [...]
}
```

**响应（非流式）**：

标准 OpenAI 格式 + 扩展字段：

```json
{
  "id": "chatcmpl-xxxx",
  "object": "chat.completion",
  "created": 1714000000,
  "model": "gpt-4o-class",
  "choices": [...],
  "usage": {
    "prompt_tokens": 123,
    "completion_tokens": 456,
    "total_tokens": 579
  },
  "llmhub_meta": {
    "provider": "volc",
    "upstream_model": "doubao-pro-32k",
    "request_id": "req_01H...",
    "billed_cents": 3.21,
    "fallback_chain": ["volc:acc_123", "dashscope:acc_456"]
  }
}
```

**响应（流式，SSE）**：

```
data: {"id":"...","choices":[{"delta":{"content":"..."}}]}

data: {"id":"...","choices":[{"delta":{"content":"..."}}]}

data: {"id":"...","usage":{...},"llmhub_meta":{...}}

data: [DONE]
```

  "input": "...",
  "voice": "llmhub-female-mandarin-warm",
  "response_format": "mp3|wav|opus|pcm",
  "speed": 1.0,

```

  "target_lang": "zh",
  "engine": "mt|llm",
  "glossary_id": "gls_xxx"

```

#### 2.1.7 `POST /v1/images/generations`（V2）

兼容 OpenAI images 协议。

#### 2.1.8 `WS /v2/translate/realtime`（端到端同传，V2）

语音→文本 或 语音→语音 同传。消息协议详见 05 号文档 §2.5。

### 2.3 错误码规范

平台统一错误码（HTTP 状态码 + JSON body）：

```json
{
  "error": {
    "type": "insufficient_balance",
    "message": "Your balance is not enough for this request.",
    "code": "LLMH_402_001",
    "request_id": "req_01H..."
  }
}
```

| HTTP | type | 说明 |
|------|------|------|
| 400 | `invalid_request` | 请求参数错误 |
| 401 | `unauthorized` | API Key 无效 |
| 402 | `insufficient_balance` | 余额不足 |
| 403 | `scope_forbidden` | Key 权限不匹配 |
| 404 | `model_not_found` | 逻辑模型不存在 |
| 409 | `model_deprecated` | 模型已下线 |
| 429 | `rate_limited` | 用户限流 |
| 500 | `internal_error` | 平台内部错误 |
| 502 | `upstream_error` | 上游调用失败 |
| 503 | `no_account_available` | 账号池暂无可用资源 |
| 504 | `upstream_timeout` | 上游超时 |

**错误响应必须携带 `request_id`**，方便用户和客服对账。

## 4. 内部服务接口（gRPC + 事件）

### 4.1 Scheduler 服务（gRPC）

```protobuf
syntax = "proto3";
package llmhub.scheduler.v1;

service SchedulerService {
    // 挑选一个可用上游账号
    rpc Pick(PickRequest) returns (PickResponse);

    // 反馈调用结果（异步也可，事件驱动优先）
    rpc Report(ReportRequest) returns (ReportResponse);

    // 标记账号状态（管理后台用）
    rpc MarkAccount(MarkAccountRequest) returns (MarkAccountResponse);

    // 查询账号快照
    rpc GetAccountState(GetAccountStateRequest) returns (AccountState);
}

message PickRequest {
    string request_id = 1;
    int64  user_id    = 2;
    string model_id   = 3;             // 逻辑模型
    string provider_hint = 4;          // 可选：强制指定 provider
    int32  estimated_tokens = 5;
    RiskLevel risk_level = 6;
    string session_key = 7;            // 会话黏性 key（通常是 user_id + conversation_id）
    repeated string exclude_account_ids = 8;  // 重试时排除已失败账号
}

enum RiskLevel { LOW = 0; MEDIUM = 1; HIGH = 2; }

message PickResponse {
    string provider_id   = 1;
    int64  account_id    = 2;
    string upstream_model = 3;
    string vault_ref     = 4;          // 凭证引用
    string endpoint_override = 5;       // 上游特殊 endpoint
    Tier   tier          = 6;
    int64  pick_token_id = 7;           // 本次挑选的追踪 ID
}

enum Tier { T1 = 0; T2 = 1; T3 = 2; T4 = 3; }

message ReportRequest {
    string request_id  = 1;
    int64  account_id  = 2;
    ReportResult result = 3;
    int32  http_status = 4;
    string error_code  = 5;
    int32  tokens_in   = 6;
    int32  tokens_out  = 7;
    int32  duration_ms = 8;
}

enum ReportResult { SUCCESS = 0; UPSTREAM_ERROR = 1; RATE_LIMITED = 2; AUTH_FAILED = 3; TIMEOUT = 4; }
```

### 4.2 Billing 服务（gRPC）

```protobuf
service BillingService {
    // 预扣费（调用前）
    rpc Freeze(FreezeRequest) returns (FreezeResponse);

    // 结算（调用成功或失败）
    rpc Settle(SettleRequest) returns (SettleResponse);

    // 释放冻结（调用失败、无扣费）
    rpc Release(ReleaseRequest) returns (ReleaseResponse);

    // 用量查询
    rpc QueryUsage(QueryUsageRequest) returns (QueryUsageResponse);
}

message FreezeRequest {
    string request_id     = 1;
    int64  user_id        = 2;
    string model_id       = 3;
    int32  estimated_input_tokens  = 4;
    int32  estimated_output_tokens = 5;
}

message FreezeResponse {
    int64  frozen_cents       = 1;
    int64  balance_after_cents = 2;
    bool   accepted           = 3;
    string reject_reason      = 4;   // insufficient / spend_cap_hit / user_suspended
}

message SettleRequest {
    string request_id       = 1;
    int32  actual_input_tokens  = 2;
    int32  actual_output_tokens = 3;
    string provider_id      = 4;
    string upstream_model   = 5;
    int64  wholesale_cents  = 6;     // 上游成本（用于毛利核算）
}
```

### 4.3 事件（NATS JetStream）

事件主题命名：`llmhub.{domain}.{action}.v1`

| Subject | Publisher | Consumer | Payload 关键字段 |
|---------|-----------|----------|------------------|
| `llmhub.call.started.v1` | api-gateway | metering | request_id, user_id, model, started_at |
| `llmhub.call.completed.v1` | api-gateway | billing, metering, scheduler | request_id, status, tokens, duration |
| `llmhub.call.failed.v1` | api-gateway | billing, scheduler, alerting | request_id, error_code, http_status |
| `llmhub.wallet.recharged.v1` | account | audit, notification | user_id, amount, channel |
| `llmhub.wallet.lowbalance.v1` | billing | notification | user_id, balance, threshold |
| `llmhub.pool.account.state_changed.v1` | scheduler | audit, alerting, admin-web (WS push) | account_id, from, to, reason |
| `llmhub.pool.account.quarantined.v1` | scheduler | alerting | account_id, provider, reason |
| `llmhub.recon.completed.v1` | billing (cron) | alerting | day, diffs_count, alert_count |
| `llmhub.user.suspicious.v1` | scheduler/gateway | alerting, user-ops | user_id, reasons[] |

**事件语义**：

- **至少一次投递** + 消费端幂等（按 request_id / unique event_id）
- **不保证顺序**，消费方需要容忍乱序（call.completed 可能先于 call.started 到达？不会，但要有超时兜底）
- **事件 payload 脱敏**：不含 prompt/completion 文本

## 5. SDK 与客户端集成

### 5.1 MVP 阶段：OpenAI SDK 直接兼容

用户无需自定义 SDK，直接用官方 OpenAI SDK：

```python
from openai import OpenAI

client = OpenAI(
    api_key="sk-llmh-xxxx",
    base_url="https://api.llmhub.io/v1"
)

resp = client.chat.completions.create(
    model="gpt-4o-class",
    messages=[{"role": "user", "content": "hello"}],
    stream=True
)
```

```javascript
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: 'sk-llmh-xxxx',
  baseURL: 'https://api.llmhub.io/v1'
});
```

### 5.2 V1 可选：官方轻量 SDK

只有以下场景才值得做自研 SDK：

- 需要暴露 LLMHub 特有能力（多供给策略、成本上限提示）
- 需要内建重试/故障切换逻辑
- 需要用户侧简单的用量上报

骨架：

```
sdk/
├── python/   # pip install llmhub
├── node/     # npm install llmhub
└── go/
```

## 6. 接口设计原则总结

1. **对外协议向已有生态兼容**（OpenAI + Anthropic），降低用户迁移成本
2. **扩展字段加前缀** `llmhub_meta`，不破坏协议兼容
3. **所有响应带 `request_id`**，用于客服/对账/调试
4. **内部服务用 gRPC + proto**，强类型 + 多语言友好
5. **跨服务状态变更走事件**，避免强耦合
6. **幂等性**：创建类接口支持 `Idempotency-Key` 头
7. **分页**：一律 cursor 分页，不用 offset（大数据量性能）
8. **时间字段**：一律 ISO 8601 / UTC，避免时区歧义
9. **金额字段**：一律用 `_cents` 后缀，整数，不用浮点
10. **版本演进**：URL 带 v1/v2；事件主题带 .v1 后缀；破坏性变更必须新版本
